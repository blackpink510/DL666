Lab02 - Dynamic Computational Graphs (solution)¶

How different are computation in PyTorch from numpy? One main difference is that PyTorch constructs a computational network as it executes. The computation network allows for backpropagation where the gradients are computed, thus allowing us to update the parameter values. In this session, we shall learn how PyTorch works under the hood.

  2. Learn how to perform backward propagation

1. Computational Graph¶

Different from numpy computations, a deep learning framework such as TensorFlow and PyTorch constructs a computational network as it runs. The computational graph is used to compute the gradient of the parameters, which in turn is used by the Gradient Descent algorithm to update the model parameters. The gradient descent algorithm is an iterative process that iterates between the following two steps:

  1. forward propagation computes the predicted output of the training sample and the loss function $L$. In the process, the computational graph is built.
  2. backward propagation computes the gradient of the parameters $\frac{dL}{dw_i}$. In the process, the computational graph is destroyed.

2. Forward Propagation¶

2.1 Building the computational graph dynamically¶

PyTorch builds the computational graph under the hood on the fly as you code. This makes PyTorch very pythonic and easy to debug. Let's see what is meant by dynamic network computation by implementing the following function.

$$out = \sum_{i=1}^M \tanh(\mathbf{W}_x\mathbf{x} + \mathbf{W}_h \mathbf{h})_i \tag{1}$$

We shall implement the function in several steps:

$$\mathbf{i2h} = \mathbf{W}_x.\mathbf{x}$$
$$\mathbf{h2h} = \mathbf{W}_h.\mathbf{h}$$
$$\mathbf{next\_h} = \mathbf{i2h} + \mathbf{h2h}$$
$$\mathbf{next\_h} = \tanh(\mathbf{next\_h})$$
$$out = \sum_{i=1}^M \mathbf{next\_h}_i$$

where:

  • the parameters $\mathbf{W}_x$ and $\mathbf{W}_h$ are random tensors of size (3,2) and (3,3), respective
  • the input $\mathbf{x}$ and $\mathbf{h}$ are random tensors of size (2,1) and (3,1), respectively.
  • the intermediate tensors $\mathbf{i2h}$, $\mathbf{h2h}$ and $\mathbf{next\_h}$ are tensors of size (3,1)
  • the output $out$ is a scalar

First, let's initialize Wh, Wx, x and h. For all of them, set its requires_grad=True to indicate that we wish to compute their gradients when we perform backpropagation later.

2.2 Building on a as needed basis¶

In the PyTorch framework, the computational graph is built on a as-needed basis where only the necessary part are constructed. For example, if we set the requires_grad only for $W_h$ and $W_x$ excluding $x$ and $h$, only a fraction of the computational graph will be constructed. It now no longer includes the nodes h and x.

In [27]:
# Only enable the gradient comptuation for Wh and Wx
Wh = torch.randn(3,3, requires_grad=True)
Wx = torch.randn(3,2, requires_grad=True)
x  = torch.randn(2,1)  # gradient of x is not needed
h  = torch.randn(3,1)  # gradient of h is not needed

i2h = Wx@x
h2h = Wh@h

next_h = i2h + h2h
next_h = next_h.tanh()
out    = next_h.sum()
In [28]:
make_dot(out)
Out[28]:

3. Backpropagation¶

In this section, we shall perform backproagation to compute the gradients at $\mathbf{W}_x$ and $\mathbf{W}_h$.

3.1 Retaining gradients of intermediate signals¶

By default, PyTorch only retains gradients for leaf nodes — tensors that were created directly and have requires_grad=True. You need to call .retain_grad() for intermediate tensors such as $\mathbf{next\_h}$, $\mathbf{i2h}$ and $\mathbf{x}$ if you want to access their gradients after backpropagation.

In [32]:
# retain the gradients of next_h, i2h and h2h
next_h.retain_grad()
i2h.retain_grad()
h2h.retain_grad()

3.2 Backpropagation¶

We perform backward propagation from the signal that we want to differentiate. In the example above, backpropagating from $out$ gives us $\frac{\partial out}{\partial \mathbf{W}_h}$ and $\frac{\partial out}{\partial \mathbf{W}_x}$.

In [34]:
# perform backpropagation
out.backward()

Notes ⚠️:

  1. In PyTorch, in-place operations (like +=, tanh_()) can break backpropagation if the original tensor value is needed for gradient computation, leading to runtime errors. In addition, leaf tensors with requires_grad=True cannot be modified in-place unless you disable gradient computation.
  2. Backpropagation destroys the computational graph where the intermediate states used to compute the gradients are freed to save memory. Therefore you cannot call backward twice.

3.3 Clearing the gradients¶

PyTorch does not overwrite the gradient each time you compute the gradient. Rather, it accumulates the gradient of the tensors that we have set requires_grad=True.

Consider the following function:

$$y = 2x$$

The gradient of $y$ is always:

$$\frac{dy}{dx} = 2$$

However, each time we perform forward and backward operation, the gradient will be accumulated in each iteration.

In [40]:
# Create the tensor
x = torch.tensor(6., requires_grad=True)
In [41]:
y = 2*x       # forward propagation
y.backward()  # backprop
print(x.grad) # dy/dx should be 2

tensor(2.)
In [42]:
y = 2*x       # forward propagation
y.backward()  # backprop
print(x.grad) # dy/dx should be 2

tensor(4.)
In [43]:
y = 2*x       # forward propagation
y.backward()  # backprop
print(x.grad) # dy/dx should be 2

tensor(6.)

To prevent the gradients from accummulating, we need to reset the gradients to zero before backpropagation through the command

x.grad.zero_()

This will allow us to get the fresh gradient in each iteration.

In [45]:
y = 2*x          # forward propagation
x.grad.zero_()   # clear the gradient
y.backward()     # backprop
print(x.grad)    # dy/dx should be 2

tensor(2.)

x.grad.zero_()   # clear the gradient
